Program Documentation Tool: javadoc

One of the tools that comes with the standard Java installation is a program called javadoc which will generate documentation similar to that for the Java API documentation.

Basically, you use a special kind of comment, which looks almost like the usual multi-line comments, to put the information you want to appear in the documentation into your program text. The documentation utility uses these comments along with the class and method signatures to generate the documentation.

Javadoc Comments

A javadoc comment is preceded by the characters /** and terminated by a */. All the text inside is considered to be part of the documentation. You can format javadoc comments by using standard HTML tags.

Javadoc comments should be placed immediately before the part of the program that they describe. Typically, you will have such a comment immediately befor each class or interface header, method signature or member variable declaration. By default the utility only shows information for public components.

Documentation Tags

There are some special tags you can use within your javadoc comments to indicate specific types of information. A partial list appears below; see the official documentation for others.
TagLocationEffect
@authorclass commentLists author(s) of code in documentation if -author option is used.
@versionclass commentRequires the -version option to be used.
@parammethod commentUsed to explain method parameters.
@returnmethod commentUsed to explain the return value of a method
@throwsmethod commentUsed to list exceptions thrown by the method.

Generating the Documentation

On onyx, you generate documentation for a group of Java classes by typing the command

javadoc -author -version -d html/ *.java

This will put the html files into a directory called html which you should create first. There will be a file called index.html which lists all the classes and one file for each class.

In Eclipse, you can use the Generate Javadoc option from the Project menu.

Example

/**
 * The Widget class represents a widget in a standard Thingamajig.
 *
 * @author Sam Stoodent
 * @version 1.0 CS121 Fall 2025
 */
public class Widget {

  /**
   * Carries out the central function of a Widget
   *
   * @param rotations number of widget rotations, positive values are clockwise turns,
   *   negative values are counterclockwise turns
   * @param pushes number of widget pushes, must be a positive integer
   * @return widget output
   * @throws InvalidInputsException
   */
  public double doWidgetStuff(double rotations, int pushes) throws InvalidInputsException {
    // regular single-line comment
    ...statements...

    /*
     * regular block comment
     */
    ...more statements...
  }
}

Utilities for other languages

A utility called CppDoc documents C++ programs using basically the same syntax as javadoc for specifying what goes into the documentation.
